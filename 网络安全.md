# 浏览器指纹
## canvas指纹
### 同一算法下受什么影响
在算法代码完全相同的情况下，生成的结果主要受以下维度的影响（这也是它能唯一标识设备的原因）：
- 操作系统（OS）：不同的操作系统（Windows、macOS、Linux）有不同的默认字体库和文字渲染引擎（如Windows的ClearType，macOS的Core Text）。
- 硬件设备（GPU与显卡驱动）：图像最终多由GPU加速渲染，不同厂商的显卡（NVIDIA、AMD、Intel）以及不同版本的显卡驱动，其图形计算和边缘抗锯齿（Anti-aliasing）的处理细节存在差异。
- 浏览器引擎：不同的浏览器（Chrome的Blink、Firefox的Gecko、Safari的WebKit）对Canvas API的底层实现细节不同。
- 屏幕设置：设备的屏幕分辨率、色彩深度（Color Depth）、设备的像素比（window.devicePixelRatio）可能会影响Canvas的默认缩放和像素映射。
- 用户个性化设置：用户安装的本地字体（影响Fallback字体选择）、系统级别的字体平滑/抗锯齿设置、浏览器是否开启了硬件加速等。
### 如何设计算法
设计一个优秀的Canvas指纹算法，核心目标是尽可能多地触发上述差异（增加信息熵）。设计步骤通常如下：
- 创建画布：在内存中动态创建一个不可见的<canvas>元素。
- 绘制背景与颜色：使用特定的颜色或渐变色填充，测试色彩空间的渲染差异。
- 绘制复杂文字：这是最重要的一步。使用常见字体（如Arial）和特殊符号（如包含Emoji或生僻字），触发操作系统的字体排版和渲染引擎。
- 触发抗锯齿（Anti-aliasing）：通过绘制倾斜的线条、圆形（Arc）或让文字产生微小的重叠，强制系统触发边缘平滑算法。
- 添加特效：使用阴影（shadowBlur）、半透明（rgba）或混合模式（globalCompositeOperation），放大GPU渲染计算的浮点数精度差异。
- 数据提取与哈希压缩：通过canvas.toDataURL()获取图像的Base64编码，或者通过getImageData()获取原始像素数组，最后通过Hash算法（如MD5、SHA-256、MurmurHash）将其转换为固定长度的短字符串。
### 设计一个最简单的算法（简洁有效）代码：
```
function getCanvasFingerprint() {
    // 1. 创建隐藏的Canvas
    const canvas = document.createElement('canvas');
    const ctx = canvas.getContext('2d');
    canvas.width = 200;
    canvas.height = 50;

    // 2. 绘制带颜色的背景矩形 (测试色彩渲染)
    ctx.fillStyle = "rgb(255, 102, 0)";
    ctx.fillRect(10, 5, 100, 20);

    // 3. 绘制重叠的半透明文字 (测试字体引擎、抗锯齿和透明度混合)
    ctx.textBaseline = "top";
    ctx.font = "16px 'Arial'";
    
    ctx.fillStyle = "rgba(0, 0, 255, 0.7)";
    ctx.fillText("Canvas Fingerprint 😃", 2, 10);
    
    ctx.fillStyle = "rgba(255, 0, 0, 0.7)";
    ctx.fillText("Canvas Fingerprint 😃", 4, 12); // 故意错开2像素，制造边缘混合抗锯齿

    // 4. 获取图像数据的Base64字符串
    const dataURL = canvas.toDataURL("image/png");

    // 5. 极简的哈希压缩算法（此处使用简单的djb2算法将超长base64转为短ID）
    let hash = 5381;
    for (let i = 0; i < dataURL.length; i++) {
        hash = ((hash << 5) + hash) + dataURL.charCodeAt(i); // hash * 33 + c
    }
    
    // 返回16进制的指纹字符串
    return (hash >>> 0).toString(16); 
}

console.log("您的Canvas指纹是:", getCanvasFingerprint());
```
## 音频指纹
### 同一算法下，音频指纹受什么影响？
- 操作系统及版本：Windows、macOS、Linux 或 Android 在处理底层数学库和音频栈时的方式不同。
- CPU 架构：不同的 CPU（如 x86 架构与 ARM 架构）在执行复杂的浮点数运算（特别是音频信号处理中的快速傅里叶变换 FFT、抗混叠滤波器等）时，精度和舍入规则可能存在极细微的差别。
- 音频硬件（声卡/音频芯片）：不同的硬件 DAC（数模转换器）或 DSP（数字信号处理器）的特性会影响最终的信号处理结果。
- 底层音频驱动程序：安装的不同声卡驱动会对音频信号进行不同的底层处理。
- 浏览器内核及音频实现引擎：Chrome (Blink/V8)、Firefox (Gecko/SpiderMonkey)、Safari (WebKit) 处理 Web Audio API 的底层 C++ 库不同。
- 系统的基础采样率：系统默认的音频采样率（如 44.1kHz 或 48kHz）有时会影响音频上下文的默认行为。
### 如何设计音频指纹算法？
设计一个高区分度且稳定的音频指纹算法，核心思想是：“生成信号 -> 复杂处理放大差异 -> 提取特征 -> 哈希计算”。
- 使用离线音频上下文（OfflineAudioContext）：
  - 千万不要直接使用 AudioContext，因为它是实时播放的，会受系统负载和设备当前状态影响导致结果不稳定。
  - OfflineAudioContext 会在内存中以最高速度直接渲染音频，不会真正发出声音，且结果具有确定性。
- 生成基础音频信号：
  - 使用 OscillatorNode（振荡器）生成波形（如正弦波、三角波等）。
- 加入复杂的音频处理节点（核心）：
  - 这是为了放大软硬件差异。通常会接入 DynamicsCompressorNode（动态压缩器）或 BiquadFilterNode（双二阶滤波器）。这些节点涉及大量复杂的浮点运算，最容易暴露出 CPU 和系统的运算差异。
- 渲染并提取数据：
  - 调用 startRendering()，获取渲染完成的 AudioBuffer。
  - 提取 getChannelData(0)（即 PCM 浮点数数组）。
- 哈希降维：
  - 将包含数万个浮点数的数组累加，或者提取特定的部分转换成字符串，最后通过 MD5 或 SHA-256 生成一个简短的字符串指纹（ID）。
### 设计一个最简单的算法（简洁有效）
```
async function getAudioFingerprint() {
    try {
        // 1. 创建离线音频上下文：单声道，44100采样率，渲染1秒 (共44100个样本)
        const offlineContext = new (window.OfflineAudioContext || window.webkitOfflineAudioContext)(1, 44100, 44100);

        // 2. 创建振荡器（生成三角波）
        const oscillator = offlineContext.createOscillator();
        oscillator.type = 'triangle';
        oscillator.frequency.value = 10000;

        // 3. 创建动态压缩器（用于引入复杂的数学运算以放大硬件差异）
        const compressor = offlineContext.createDynamicsCompressor();
        compressor.threshold.value = -50;
        compressor.knee.value = 40;
        compressor.ratio.value = 12;
        compressor.reduction.value = -20;
        compressor.attack.value = 0;
        compressor.release.value = 0.25;

        // 4. 连接节点：振荡器 -> 压缩器 -> 目的地
        oscillator.connect(compressor);
        compressor.connect(offlineContext.destination);

        // 5. 启动振荡器并开始渲染
        oscillator.start(0);
        const audioBuffer = await offlineContext.startRendering();

        // 6. 提取音频数据（Float32Array）
        const pcmData = audioBuffer.getChannelData(0);

        // 7. 计算简易 Hash (将浮点数进行数学运算累加)
        let fingerprint = 0;
        for (let i = 0; i < pcmData.length; i++) {
            fingerprint += Math.abs(pcmData[i]);
        }

        return fingerprint.toString(); 
        // 实际工程中，通常会将结果传入 MD5/SHA-1 获得一个固定长度的字符串
    } catch (e) {
        return "Audio_Fingerprint_Not_Supported";
    }
}

// 运行测试
getAudioFingerprint().then(fp => console.log("您的音频指纹是:", fp));
```
## webgl指纹
### 同一 WebGL 指纹算法下，受什么影响？
- 硬件层（最核心影响）：
  - 显卡（GPU）型号与架构：不同的 GPU 处理浮点数运算（IEEE 754 标准的实现细节）、三角函数计算的精度和取整规则不同。
  - 硬件性能参数：最大纹理尺寸、最大视口维度、最大各向异性过滤等物理限制。
- 软件与驱动层：
  - 显卡驱动程序及其版本：驱动程序负责将 WebGL 指令翻译为显卡能懂的机器码，不同版本的驱动可能会采用不同的着色器编译优化策略。
- 操作系统（OS）：不同的 OS 底层调用的图形 API 不同。例如，Windows 走 DirectX（Angle 转换），macOS 走 Metal，Linux 走 OpenGL/Vulkan。
- 浏览器层：
  - 浏览器内核及版本：Chrome (Blink) 和 Firefox (Gecko) 对 WebGL 规范的具体实现和安全限制有所不同。
  - 抗锯齿（Anti-aliasing）算法：浏览器和底层系统结合时，默认的边缘平滑处理策略不同，会导致渲染出的图像边缘像素颜色值产生细微差异。
### 如何设计 WebGL 指纹算法？
设计一个完备的 WebGL 指纹算法，通常分为数据采集和指纹哈希两步。核心思路是从“元数据”和“图像渲染差异”两个方面入手：
- 元数据提取（Metadata）：
  - 获取浏览器的 WebGL 基础能力参数。例如：MAX_TEXTURE_SIZE（最大纹理尺寸）、MAX_RENDERBUFFER_SIZE 等。
  - 通过 WEBGL_debug_renderer_info 扩展，获取最关键的 GPU 供应商（Vendor） 和 渲染器型号（Renderer）（例如：NVIDIA GeForce RTX 4090/PCIe/SSE2）。
- 渲染图像提取（Canvas/WebGL Rendering）：
  - 在隐藏的 Canvas 上初始化 WebGL 上下文。
  - 编写简单的顶点着色器（Vertex Shader）和片段着色器（Fragment Shader）。
  - 绘制一个带有渐变色、光照或复杂多边形的 3D 图形（通常包含特定角度的旋转）。
  - 因为前文提到的浮点精度和抗锯齿差异，不同设备渲染出的这张图片，在某些像素的 RGB 值上会差个 1 或 2。
  - 使用 canvas.toDataURL() 或 gl.readPixels() 将图像转化为 Base64 字符串或像素数组。
- 计算哈希值（Hashing）：
  - 将上述所有参数和图像字符串拼接在一起。
  - 使用哈希算法（如 MurmurHash3、SHA-256 或简单的 DJB2）将其转换为固定长度的字符串，即为该设备的 WebGL 指纹。
### 设计一个最简单的 WebGL 指纹算法（简洁有效）
为了“简洁有效”，我们不需要写几百行的着色器代码画 3D 模型。最简算法只需提取最核心的显卡型号以及基础的渲染背景差异即可。
```
function getSimpleWebGLFingerprint() {
    // 1. 创建隐藏的 canvas
    const canvas = document.createElement('canvas');
    const gl = canvas.getContext('webgl') || canvas.getContext('experimental-webgl');
    if (!gl) return "WebGL_Not_Supported";

    let fingerprintData = "";

    // 2. 提取极其关键的显卡元数据
    const debugInfo = gl.getExtension('WEBGL_debug_renderer_info');
    if (debugInfo) {
        fingerprintData += gl.getParameter(debugInfo.UNMASKED_VENDOR_WEBGL) + "|";
        fingerprintData += gl.getParameter(debugInfo.UNMASKED_RENDERER_WEBGL) + "|";
    }

    // 3. 提取基础上下文参数（举例两个最具代表性的限制）
    fingerprintData += gl.getParameter(gl.MAX_TEXTURE_SIZE) + "|";
    fingerprintData += gl.getParameter(gl.MAX_VIEWPORT_DIMS).join(',') + "|";

    // 4. 触发极简渲染差异：利用特定的浮点数颜色清空画布
    // 不同 GPU 对带小数的颜色值的位深转换和精度处理有极其微小的差异
    gl.clearColor(0.123, 0.456, 0.789, 1.0);
    gl.clear(gl.COLOR_BUFFER_BIT);

    // 提取画布图像的 Base64 编码
    fingerprintData += canvas.toDataURL();

    // 5. 极简的哈希函数 (DJB2) 压缩长字符串
    let hash = 5381;
    for (let i = 0; i < fingerprintData.length; i++) {
        hash = ((hash << 5) + hash) + fingerprintData.charCodeAt(i); /* hash * 33 + c */
    }
    
    // 返回最终的 16进制指纹字符串
    return hash.toString(16);
}

console.log("当前设备的 WebGL 指纹: ", getSimpleWebGLFingerprint());
```
## webgpu指纹
### 同一算法下受什么影响？
- 底层硬件（GPU）架构差异：不同厂商（NVIDIA、AMD、Intel、Apple Silicon）甚至同厂商的不同架构（如 RTX 30系 vs 40系），其算术逻辑单元（ALU）处理浮点数运算（如 sin, cos, tan, fract，尤其是超越函数）时的底层硬件实现方式不同，会导致微小的精度截断或舍入差异。
- 显卡驱动程序版本：驱动程序负责将 WebGPU 的着色器语言（WGSL）编译为 GPU 的机器码。驱动的更新可能会改变编译器的优化策略，从而导致计算结果发生微小变化。
- 操作系统与图形后端：浏览器在不同 OS 上使用不同的底层图形 API（Windows 用 Direct3D 12，macOS 用 Metal，Linux 用 Vulkan）。WebGPU 在浏览器的底层实现（如 Chrome 的 Dawn 或 Firefox 的 wgpu）会将 WGSL 翻译为对应后端的着色器语言（HLSL/MSL/SPIR-V），翻译过程的差异会影响结果。
- 浏览器版本：浏览器引擎对 WebGPU 标准的实现更新（例如暴露的 limits 参数的增减、特性的支持情况）。
### 如何设计算法？
设计 WebGPU 指纹算法通常包含两个主要部分，将静态硬件限制与动态计算特征结合起来：
- 静态信息收集（Adapter Info & Limits & Features）：
  - 获取 GPU 供应商（Vendor）、架构（Architecture）。
  - 遍历并提取 navigator.gpu.requestAdapter() 返回的 limits（如 maxTextureDimension2D, maxComputeWorkgroupsPerDimension 等），这些参数严格受制于硬件和浏览器策略。
  - 获取支持的 features 集合（如纹理压缩格式 texture-compression-bc 等）。
- 动态计算特征（Shader 精度指纹）：
  - 核心思想：利用 Compute Shader（计算着色器）执行一段极其复杂的浮点数数学运算。
  - 避免简单的加减乘除（这些通常遵守 IEEE 754 标准，全平台一致），而应大量使用超越函数（如三角函数、对数函数、指数函数）和取小数操作（fract）。
  - 将计算结果写入显存（Buffer），再读取回内存（JavaScript 端）。
- 哈希生成：将静态信息与动态计算的浮点数数组序列化，使用 SHA-256 或 MurmurHash 生成最终的指纹字符串。
### 设计一个最简单的算法（简洁有效）
相比渲染一个复杂的图形（Pipeline 繁琐），使用 WebGPU 的 Compute Shader（计算着色器） 是最简洁高效的方案。以下是一个开箱即用的前端算法示例：
```
async function getWebGPUFingerprint() {
    if (!navigator.gpu) return "WebGPU not supported";

    try {
        const adapter = await navigator.gpu.requestAdapter();
        if (!adapter) return "No adapter found";
        const device = await adapter.requestDevice();

        // 1. 收集静态 Limits（硬件限制）
        const limits = {};
        for (const key in adapter.limits) {
            limits[key] = adapter.limits[key];
        }

        // 2. 动态计算特征：执行复杂的浮点运算
        // 使用 WGSL 编写计算着色器，进行三角函数和对数运算
        const wgslCode = `
            @group(0) @binding(0) var<storage, read_write> output: array<f32>;
            
            @compute @workgroup_size(1)
            fn main(@builtin(global_invocation_id) global_id : vec3<u32>) {
                let x = f32(global_id.x) + 1.123;
                // 复杂的组合运算，最容易体现硬件浮点数精度差异
                let value = tan(x * 12.345) * log(x + 50.0) + fract(sin(x) * 43758.5453123);
                output[global_id.x] = value;
            }
        `;

        const shaderModule = device.createShaderModule({ code: wgslCode });
        const computePipeline = device.createComputePipeline({
            layout: 'auto',
            compute: { module: shaderModule, entryPoint: 'main' }
        });

        // 创建缓冲区（计算4个浮点数，共16字节）
        const numFloats = 4;
        const byteSize = numFloats * 4;
        const workBuffer = device.createBuffer({
            size: byteSize,
            usage: GPUBufferUsage.STORAGE | GPUBufferUsage.COPY_SRC
        });
        const readBuffer = device.createBuffer({
            size: byteSize,
            usage: GPUBufferUsage.COPY_DST | GPUBufferUsage.MAP_READ
        });

        // 录制并提交命令
        const commandEncoder = device.createCommandEncoder();
        const passEncoder = commandEncoder.beginComputePass();
        passEncoder.setPipeline(computePipeline);
        passEncoder.setBindGroup(0, device.createBindGroup({
            layout: computePipeline.getBindGroupLayout(0),
            entries:[{ binding: 0, resource: { buffer: workBuffer } }]
        }));
        passEncoder.dispatchWorkgroups(numFloats); // 派发4个工作组
        passEncoder.end();
        commandEncoder.copyBufferToBuffer(workBuffer, 0, readBuffer, 0, byteSize);
        device.queue.submit([commandEncoder.finish()]);

        // 3. 读取计算结果
        await readBuffer.mapAsync(GPUMapMode.READ);
        const resultArray = new Float32Array(readBuffer.getMappedRange());
        const dynamicMathStr = Array.from(resultArray).join(',');
        readBuffer.unmap();

        // 4. 合并并生成简单的 Hash (此处简单演示，实际可用 SHA-256)
        const rawData = JSON.stringify({ limits, dynamicMathStr });
        let hash = 0;
        for (let i = 0; i < rawData.length; i++) {
            hash = ((hash << 5) - hash) + rawData.charCodeAt(i);
            hash |= 0; 
        }

        return `Hash: ${hash} \nMathResult: ${dynamicMathStr}`;
    } catch (e) {
        return "Error: " + e.message;
    }
}

getWebGPUFingerprint().then(console.log);
```
